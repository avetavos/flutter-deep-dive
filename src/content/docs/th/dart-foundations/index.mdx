## Flutter คือ Dart

ก่อนจะแตะ widget ตัวแรก เราเริ่มที่ภาษา เพราะ **Flutter ทุกบรรทัดที่คุณเขียนคือ Dart** รูปทรงของ framework — widget ที่ immutable, `const` constructor, `async`/`await` ใน build logic, mixin อย่าง `SingleTickerProviderStateMixin` — ล้วนเป็นผลโดยตรงจาก feature ของ Dart พอเข้าใจภาษาดีแล้ว Flutter ก็เลิกเป็นกองคาถาวิเศษ

export const mapDiagram = `flowchart LR
  ns["Null safety"] --> w["widget props ที่ปลอดภัย,\noptional params"]
  cc["const constructors"] --> perf["rebuild widget ราคาถูก"]
  mx["mixins"] --> st["State mixins\n(ticker, observer)"]
  as["async / Stream"] --> ui["FutureBuilder,\nStreamBuilder"]`;

- **Null safety** กำหนดว่าคุณ model widget property และข้อมูลที่เป็น optional อย่างไร
- **`const` constructor** คือ lever ที่ใหญ่ที่สุดของ performance ใน Flutter — widget ที่เป็น `const` จะถูกข้ามตอน rebuild
- **Mixin** คือวิธีที่ Flutter เพิ่มความสามารถให้ `State` class ของคุณ
- **Async และ Stream** คือวิธีที่ UI คุยกับ network, database และเวลา

<Callout title="mental shift">
มาจาก framework อื่นคุณอาจอยากกระโดดไป widget เลย อย่าเพิ่ง ใช้เวลาสิบนาทีกับ null safety และ `const` constructor ของ Dart จะช่วยประหยัดเวลาสับสนหลายชั่วโมงทีหลัง — "ทำไมมันไม่ rebuild?", "ทำไมค่านี้ null?", "ทำไม list ถึง jank?" เกือบทั้งหมดคือคำถามเรื่อง Dart ที่ใส่ชุด Flutter มา
</Callout>
