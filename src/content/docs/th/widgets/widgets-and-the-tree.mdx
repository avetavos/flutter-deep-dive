## composition ไม่ใช่ inheritance

Flutter สร้าง UI ด้วยการ **nest** widget ไม่ใช่ subclass มัน คุณไม่ extend `Button` เพื่อเพิ่ม padding — คุณ *wrap* มันด้วย `Padding` หนึ่งหน้าจอคือ tree ที่ลึกของ widget เล็ก ๆ ที่ทำหน้าที่เดียว ประกอบเข้าด้วยกัน

```dart
Widget build(BuildContext context) {
  return Center(
    child: Padding(
      padding: const EdgeInsets.all(16),
      child: Column(
        children: [
          const Text('Hello'),
          ElevatedButton(onPressed: () {}, child: const Text('Tap')),
        ],
      ),
    ),
  );
}
```

นี่คือ composition over inheritance แบบสุดโต่ง: widget เล็ก ๆ จำนวนมาก แต่ละตัวทำสิ่งเดียว nest กันจนเป็นภาพรวม `Center`, `Padding`, `Column` ไม่ใช่ของประดับ — พวกมัน *คือ* layout

## สาม tree

นี่คือ model ที่อธิบายเกือบทุกอย่างเกี่ยวกับ behavior และ performance ของ Flutter มันไม่ได้มี tree เดียว — มันมี **สาม tree** ที่ถูก sync ให้ตรงกัน:

- **Widget** — description ที่ immutable ที่คุณเขียน rebuild ตลอดและทิ้งไป ถูก
- **Element** — *living instance* ของ widget ณ ตำแหน่งหนึ่งใน tree มันอยู่ต่อเนื่องข้าม rebuild, ถือ `State` ของ stateful widget และตัดสินว่า widget ใหม่จะ update มันได้หรือต้องแทนที่
- **RenderObject** — ทำ layout, paint และ hit-testing จริง นี่คือ layer ที่แพง และ Flutter พยายามอย่างหนักที่จะนำมันกลับมาใช้ซ้ำ

## ทำไม rebuild ถึงยังถูก

เมื่อคุณ rebuild Flutter จะเดิน widget tree ใหม่และเทียบ widget ใหม่แต่ละตัวกับ element ที่มีอยู่ ถ้า widget ใหม่มี **type และ key เดียวกัน** กับตัวเก่า element จะถูก *update ในที่เดิม* — render object ถูกใช้ซ้ำ อัปเดตเฉพาะ property ที่เปลี่ยน เฉพาะเมื่อ type หรือ key ต่างเท่านั้น Flutter จึงรื้อ element กับ render object แล้ว build ใหม่

```dart
// การ rebuild สร้าง Text widget ใหม่ทุกครั้ง...
Text(counter.toString())
// ...แต่ TextElement และ RenderParagraph ข้างล่างถูกใช้ซ้ำ
// มีแค่ string ที่เปลี่ยนถูกส่งลงไป นี่คือเหตุผลที่ setState ถูก
```

การ reconciliation นี้ — widget ใหม่ที่ immutable ถูก diff กับ element ที่อยู่ต่อเนื่อง — คือห้องเครื่องของ Flutter Key (บทเรียนถัดไป) มีไว้เพื่อควบคุมมันเมื่อการ match ด้วย type แบบ default ทำผิด

<Callout title="สรุปหนึ่งบรรทัด">
Flutter สร้าง UI ด้วยการ **compose** widget เล็ก ๆ ไม่ใช่ subclass ข้างล่างมีสาม tree ทำงานคู่ขนาน: **widget** ที่ทิ้งได้ (description), **element** ที่อยู่ต่อเนื่อง (living instance ที่ถือ state) และ **render object** ที่แพง (layout และ paint) การ rebuild ถูกเพราะ Flutter นำ element และ render object กลับมาใช้ซ้ำทุกที่ที่ widget ใหม่ตรงกับตัวเก่าด้วย type และ key
</Callout>
